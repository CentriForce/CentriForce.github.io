第一个博客

这是我第一个可以用来编辑上传文本的博客。我希望我以后可以通过在这个博客上上传文章，来加深我对陌生知识的理解与记忆。同时方便我以后在需要用到这些知识的时候可以随时找到。

一、如何使用 Hexo + GitHub 搭建一个博客

1.注册 GitHub 账号

  • 点击链接注册（可能需要科学上网）。
  • 完成注册之后，点击右上角+添加一个New repository。
  • 编辑Repository name
    • Repository name的格式为GitHub 用户名.github.io。例如Jack的Repository name要输入Jack.github.io
  • 点击Create repository

这样，我们在Github上的静态托管存储库就建立好了。我们只需要将博客的文件放在上面托管，就可以通过GitHub 用户名.github.io这个网址来访问了（慢但是似乎不需要翻墙）。

2.配置本地连接 GitHub

静态托管网页的优点是不需要租赁一个单独的服务器，但是缺点是无法在线上直接编辑并发表博客。我们需要在本地同步云端的文件，即本地编辑->云端同步。

所以，我们需要下载node.js和git两个软件来进行同步。

  • 安装node.js和git

  • 新建一个文件夹准备存放 Hexo 的程序文件，如 Hexo-Blog

  • 打开文件夹，对空白处右键，选择Git Bash Here

  • 在新出现的命令行窗口中输入以下内容

      git config --global user.name "GitHub 用户名"
  git config --global user.email "GitHub 邮箱"
  ssh-keygen -t rsa -C "GitHub 邮箱"

    然后不断地按回车直到结束。

  • 进入C:\Users\用户名\.ssh目录（要勾选显示“隐藏的项目”），用记事本打开公钥 id_rsa.pub 文件并复制里面的内容

  • 登陆GitHub，进入Setting页面，选择左边栏的SSH and GPG keys，点击New SSH key

  • Title 随便取个名字，粘贴复制的 id_rsa.pub 内容到 Key 中，点击 Add SSH key 完成添加

  • 返回之前的新建文件夹，对空白处右键，打开Git Bash

  • 输入ssh -T git@github.com，当出现 Are you sure you want to continue connecting (yes/no)?时，输入 yes 回车确认。

  • 若显示Hi "GitHub 用户名"! You've successfully……即连接成功。

3.安装 Hexo 程序和本地预览博客

我们需要自己安装一个博客程序来帮助我们在本地管理博客。这样的话，我们只需要写博客，关于网页上博客内容排版，搜索博客等功能可以自动实现。

  • 打开文件夹，对空白处右键，选择Git Bash Here

  • 输入npm install -g hexo-cli

  • 等hexo安装完成之后，输入

      hexo init
  npm install

  • 完成后，整个博客的雏形已经出现了。我们可以启动本地服务器来预览我们的博客。输入

      hexo g
  hexo s

  • 访问http://localhost:4000，出现 Hexo 默认页面，本地博客安装成功！

4. 部署 Hexo 到 GitHub Pages

本地博客测试成功后，就是上传到 GitHub 进行部署，使其能够在网络上访问。

  • 安装 hexo-deployer-git：

      npm install hexo-deployer-git --save

  • 修改新建文件夹中的_config.yml文件末尾的Deployment部分，修改成如下：

      deploy:
      type: git  
      repository: git@github.com:用户名/用户名.github.io.git
      branch: master

  • 在Git Bash中输入hexo d以将网站上传部署到 GitHub Pages（需要科学上网）

这时，当网站上传部署完成后，访问我们的 GitHub 域名https://用户名.github.io就可以看到我们的博客了。

5.套模板美化博客与个性化

Hexo 官方提供了很多博客爱好者为 Hexo 制作的博客主题。即使不会使用 HTML5+CSS 设计网页，也可以通过将他人制作好的主题复制到本地，然后再部署到GitHub来美化自己的博客。

  • 在 Themes | Hexo里选择一个喜欢的主题
  • 点击主题的标题，进入主题的GitHub页面
  • 不同的主题复杂程度不同，例如主题NexT简单，主题Arknights复杂。最好根据主题的README.md描述来应用主题

除此之外，你肯定也想让自己的博客有自己的名字，图标等等。这些网站设置全部都在文件夹根目录下的_config.yml 文件中，你可以参考官方文档按需要编辑。应用主题之后，可能还需要根据主题的要求再更改一些其他的内容。

hexo new "My New Post"
